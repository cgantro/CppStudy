## 함수 타입, 함수 포인터, 함수 객체

#### [코드](./code/function.cpp)

함수 자체는 객체가 아니다. 함수 포인터는 객체다.

---

## 함수 포인터

반환 타입과 파라미터 타입으로 함수 포인터 타입이 결정된다.

```cpp
int add(int a, int b) { return a + b; }

int (*fp)(int, int) = add;  // 함수 포인터 선언
int result = fp(1, 2);      // 3

// using으로 가독성을 높일 수 있다
using BinaryOp = int(*)(int, int);
BinaryOp op = add;
```

---

## 함수 객체 (Functor)

`operator()`를 오버로드한 클래스. 상태를 가질 수 있어 함수 포인터보다 유연하다.

```cpp
struct Adder {
    int base;
    int operator()(int x) const { return base + x; }
};

Adder add5{5};
add5(3);  // 8
add5(10); // 15
```

---

## 람다

람다는 함수 객체(익명 클래스)를 컴파일러가 생성하는 문법적 편의 기능이다.

```cpp
int base = 10;
auto addBase = [base](int x) {
    return base + x;
};
addBase(5);  // 15
```

### 캡처 방식

```cpp
int a = 1, b = 2;

auto f1 = [a]()    { return a; };        // 값 복사 캡처
auto f2 = [&b]()   { return b; };        // 참조 캡처 (b가 소멸하면 UB)
auto f3 = [=]()    { return a + b; };    // 모든 변수 값 복사
auto f4 = [&]()    { return a + b; };    // 모든 변수 참조
auto f5 = [=, &b]() { return a + b; };  // 기본은 복사, b만 참조
```

참조 캡처는 캡처한 변수가 람다보다 먼저 소멸하면 댕글링 참조가 된다. 람다를 저장해 나중에 사용할 때는 값 복사 캡처가 안전하다.

### 현대 C++에서의 선택 기준

| 상황 | 권장 |
|---|---|
| 단순 콜백 (즉시 사용) | 람다 |
| 타입을 저장·전달해야 함 | `std::function` |
| 상태가 필요한 재사용 객체 | 함수 객체 |
| 함수 포인터가 요구되는 C API | 캡처 없는 람다 또는 함수 포인터 |
