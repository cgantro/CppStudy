## 템플릿

#### [코드](./code/template.cpp)

타입을 매개변수로 받는 컴파일 타임 메커니즘이다.  
함수 호출 시 인자 타입을 보고 템플릿 파라미터를 추론(deduction)한다.

```cpp
template<typename T>
T maxValue(T a, T b) {
    return a > b ? a : b;
}

int    a = maxValue(1, 2);       // T = int
double b = maxValue(1.5, 2.5);  // T = double

// 서로 다른 타입을 허용하려면 반환 타입을 별도로 추론한다
template<typename T, typename U>
auto add(T a, U b) {
    return a + b;
}

auto x = add(1, 2.5);  // double (암시적 형변환)
```

### 템플릿 특수화

특정 타입에 대해 별도 구현을 제공한다.

```cpp
template <typename T>
struct TypeName {
    static void print() { std::cout << "unknown\n"; }
};

template <>
struct TypeName<int> {
    static void print() { std::cout << "int\n"; }
};

TypeName<double>::print();  // unknown
TypeName<int>::print();     // int
```

### 타입 특성 검사 (`<type_traits>`)

컴파일 타임에 타입의 성질을 조회한다.

```cpp
template <typename T>
void printCategory() {
    if constexpr (std::is_integral_v<T>)
        std::cout << "Integral\n";
    else if constexpr (std::is_floating_point_v<T>)
        std::cout << "floating point\n";
    else
        std::cout << "other\n";
}

printCategory<int>();         // Integral
printCategory<double>();      // floating point
printCategory<std::string>(); // other
```

`if constexpr`는 조건이 false인 분기를 컴파일 자체를 하지 않는다. 일반 `if`와 달리 해당 타입에서 존재하지 않는 연산을 써도 에러가 나지 않는다.

## Concepts: 타입에 조건 붙이기

#### [Concepts 코드](./code/concepts.cpp)

C++20의 concept은 **템플릿 인자가 만족해야 할 조건**을 명시하는 방법이다.  
조건을 위반하면 내부 구현 오류 대신 명확한 컴파일 에러가 발생한다.

### concept 정의

```cpp
#include <concepts>

// 직접 concept 정의
template <typename T>
concept Numeric = std::is_integral_v<T> || std::is_floating_point_v<T>;

// 표준 라이브러리 concept 활용
// std::integral, std::floating_point, std::same_as, std::convertible_to 등
```

### 사용 방법 3가지

```cpp
// 1. template 파라미터에 직접 사용
template <Numeric T>
T square(T x) { return x * x; }

// 2. requires 절
template <typename T>
    requires Numeric<T>
T square(T x) { return x * x; }

// 3. abbreviated function template (가장 간결)
auto square(Numeric auto x) { return x * x; }
```

### requires 표현식으로 상세 조건 작성

```cpp
template <typename T>
concept Printable = requires(T t) {
    { std::cout << t } -> std::same_as<std::ostream&>;  // << 연산자가 있어야 함
};

template <typename T>
concept Container = requires(T c) {
    c.begin();           // begin() 멤버가 있어야 함
    c.end();             // end() 멤버가 있어야 함
    c.size();            // size() 멤버가 있어야 함
    typename T::value_type;  // value_type 중첩 타입이 있어야 함
};
```

### Concepts가 없을 때와 비교

```cpp
// Concepts 없음: 조건 위반 시 긴 내부 오류 메시지
template <typename T>
T add(T a, T b) { return a + b; }

// Concepts 있음: 즉시 명확한 에러
template <typename T>
    requires std::integral<T> || std::floating_point<T>
T add(T a, T b) { return a + b; }

// add("hello", "world") — "T does not satisfy integral or floating_point" 로 즉시 에러
```
