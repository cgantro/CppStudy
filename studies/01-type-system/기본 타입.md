## 정수 타입

```cpp
short, int, long, long long, unsigned int ...;
```

정수 타입은 크기가 플랫폼에 따라 달라진다.

- 정확한 크기가 필요하면 `<cstdint>`를 사용한다
- `sizeof`로 실제 바이트 크기를 확인할 수 있다

```cpp
#include <iostream>

std::cout << sizeof(int) << '\n';   // 보통 4
std::cout << sizeof(long) << '\n';  // Windows: 4, Linux/macOS: 8
```

```cpp
#include <cstdint>

std::int32_t id = 100;       // 정확히 32비트 부호 있는 정수
std::uint64_t mask = 0xff;   // 정확히 64비트 부호 없는 정수

// using 별칭은 강한 타입이 아니다 — 같은 정수 타입으로 취급됨
using UserId = std::uint64_t;

// 강한 타입을 위해선 구조체로 래핑한다
struct StrongUserId {
    std::uint64_t value;
};
```

### 부동소수점 타입

```cpp
float f = 0.1f;
double d = 0.1;
long double ld = 0.1L;
```

정확한 10진수 표현이 아니다.

- [부동소수점 코드](./code/부동소수점.cpp)
- 따라서 비교할 때, 보통 오차 허용 범위(epsilon)를 사용한다

```cpp
#include <cmath>
#include <limits>

bool nearly_equal(double a, double b) {
    return std::abs(a - b) < std::numeric_limits<double>::epsilon();
}
```

### bool, char, std::byte

바이너리 데이터를 표현 시에, `char`보다 `std::byte`가 의도가 명백하다.

```cpp
std::vector<std::byte> buffer;
```

- `char`는 문자 또는 raw byte 양쪽에 쓰일 수 있어 의미가 흐려질 수 있음
- `std::byte`는 비트 연산만 허용하여 의도를 명확히 한다
- `bool`은 `true`/`false`지만 내부 크기는 구현 정의 (보통 1바이트)
