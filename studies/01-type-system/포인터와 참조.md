## 포인터

포인터는 객체의 주소를 저장하는 타입

```cpp
int x = 10;
int *p = &x;

*p = 20;

// 포인터는 null이 될 수 있고, 다른 대상을 가리킬 수 있음.

int *p = nullptr;

// 포인터는 소유권을 표현하지 않음

void observe(int *p){
    if(p != nullptr){
        // 관찰만 하는가
        // 소유권을 받는가
    }
}

// 소유권이 있으면 스마트 포인터 사용

std::unique_ptr<int> makeValue(){
    return std::make_unique<int>(10);
}
```

## 참조

참조는 기존 객체의 별칭

```cpp
int x = 10;
int &r = x;

r = 20; // x도 20
```

참조는 일반적으로 null 불가

- 초기화 후 다른 객체를 참조하도록 바꿀 수 없음

```cpp
int a = 10;
int b= 20;
int &r = a;
r = b;
// r이 b를 가리키게 되는 것이 아님
// a에 b값을 대입하는 것임
```

## lval 참조와 rval 참조

```cpp
void f(int& x){
    // lval 받음
}

void g(int&& x){
    // rval 받음
}

int main(){
    int a = 10;
    f(a);
    // f(10) -> 에러
    g(10);
    // g(a) -> 에러
}
```

rval ref은 이동 생성자, 이동 대입, perfect forwarding에서 핵심
