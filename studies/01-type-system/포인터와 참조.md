## 포인터

포인터는 객체의 주소를 저장하는 타입이다.

```cpp
int x = 10;
int* p = &x;
*p = 20;  // x가 20이 됨

int* q = nullptr;  // null 포인터
```

포인터는 소유권을 표현하지 않는다. 함수가 포인터를 받을 때 소유권이 있는지 없는지 코드만 봐서는 알 수 없다.

```cpp
void observe(int* p) {
    // 관찰만 하는가? 소유권을 받는가?
    // 포인터만 봐서는 알 수 없다
}

// 소유권이 있으면 스마트 포인터로 명시한다
std::unique_ptr<int> makeValue() {
    return std::make_unique<int>(10);
}
```

### 댕글링 포인터 (Dangling Pointer)

가리키는 객체가 이미 소멸한 포인터. 역참조하면 미정의 동작(UB)이다.

```cpp
int* p = nullptr;
{
    int x = 10;
    p = &x;
}  // x가 소멸
*p;  // UB — 댕글링 포인터 역참조
```

## 참조

참조는 기존 객체의 별칭이다. 선언 시 반드시 초기화해야 하고, 초기화 후에 다른 객체로 재바인딩할 수 없다.

```cpp
int x = 10;
int& r = x;
r = 20;  // x도 20이 됨

int a = 10, b = 20;
int& r = a;
r = b;  // r이 b를 가리키는 것이 아니라 a에 b 값을 대입하는 것
```

참조는 일반적으로 null이 될 수 없다. 따라서 참조를 받는 함수는 null 체크가 필요 없다.

## lvalue 참조와 rvalue 참조

- **lvalue 참조(`T&`)** : 이름이 있고 주소를 취할 수 있는 값
- **rvalue 참조(`T&&`)** : 임시 객체, 이동 후 더 이상 필요 없는 값

```cpp
void f(int& x)  {}  // lvalue만 받음
void g(int&& x) {}  // rvalue만 받음

int a = 10;
f(a);    // ok
// f(10) — 에러: 10은 rvalue
g(10);   // ok
// g(a)  — 에러: a는 lvalue
```

### `std::move`와의 관계

`std::move`는 lvalue를 rvalue로 캐스팅한다. 실제로 이동이 일어나는 것이 아니라, "이 값을 이동해도 된다"는 신호다.

```cpp
std::string s = "hello";
std::string t = std::move(s);  // s의 소유권을 t로 이동
// s는 이후 유효하지만 내용이 비어 있는 상태 (moved-from state)
```

rvalue 참조는 이동 생성자, 이동 대입, perfect forwarding의 핵심이다.
