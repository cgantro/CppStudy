# 타입 변환

크게 세 가지

1. 표준 변환
2. 사용자 정의 변환
3. 명시적 캐스트

## 암시적 변환

```cpp
int i = 10;
double d = i; // int -> double
```

암시적 변환은 버그를 만들 수 있기에, 주의해야 한다.

## narrowing conversion

중괄호 초기화는 narrowing을 막아준다

- 정밀도 손실 등 잠재적으로 안전하지 않은 데이터 형변환

```cpp
int a = 3.14; // 컴파일 가능. 값 손실.
// int b{3.14}; // 컴파일 에러

// 타입 안정성을 높이기 위해 중괄호 초기화
struct Packet{
    std::uint32_t size;
}

Packet p{1024};
```

## C++ 캐스트 4종

#### `static_cast`

컴파일 타임에 비교적으로 명확히 변환을 해준다

```cpp
double d = 3.14;
int i = static_cast<int>(d);
```

#### `reinterpret_cast`

비트 수준에서의 재해석

- 위험함

```cpp
#include <cstdint>

int x = 10;
std::uintptr_t addr = reinterpret_cast<std::uintptr_t>(&x);
// 타입 시스템의 보호를 크게 우회하므로, 시스템 코드, FFI, 네트워크 직렬화 등 제한된 영역에서만 사용해야 한다.
```

#### `const_cast`

`const` 속성을 제거하거나 추가한다.

```cpp
void legacyApi(char* p){}

void call(const char* test){
    legacyApi(const_cast<char*>(test);
}
```

#### `dynamic_cast`

다형 타입에서 런타임 타입 확인

```cpp
#include <iostream>

struct Base{
    virtual ~Base() = default;
};

struct Derived : Base{
    void hello(){
        std::cout << "Derived\n";
    }
};

void call(Base* b){
    if(auto *d = dynamic_cast<Derived*>(b)){
        d->hello();
    }
}
```

## 정적 타입과 동적 타입

#### [코드](./code/정적_동적타입.cpp.cpp)

C++은 정적 타입 언어

- 변수와 표현식의 타입은 컴파일 타임에 결정됨

```cpp
Base* p = new Derived{};

// 정적타입 -> Base*
// 동적 타입 -> 실제 객체인 Derived*
```

다형성을 사용하려면 기반 클래스에 virtual 함수를 선언해야 한다.

- virtual이 없으면 정적 타입 기준으로 호출 된다.
