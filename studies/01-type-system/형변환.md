# 타입 변환

크게 세 가지

1. 표준 변환 (암시적)
2. 사용자 정의 변환
3. 명시적 캐스트

---

## 암시적 변환

```cpp
int i = 10;
double d = i;  // int → double
```

암시적 변환은 버그를 만들 수 있기에 주의해야 한다.

---

## Narrowing Conversion

정밀도 손실 등 잠재적으로 안전하지 않은 암시적 변환이다. 중괄호 초기화는 이를 컴파일 에러로 막아준다.

```cpp
int a = 3.14;    // 컴파일 가능 — 값 손실 (a = 3)
// int b{3.14};  // 컴파일 에러 — narrowing 차단

struct Packet {
    std::uint32_t size;
};
Packet p{1024};  // 안전
```

## 사용자 정의 변환

### 변환 생성자

인자가 하나인 생성자는 암시적 변환 경로가 된다.

```cpp
struct Meter {
    explicit Meter(double v) : value(v) {}
    double value;
};

// explicit 없으면 암시적 변환 허용
// explicit 있으면 아래처럼 명시적으로만 생성 가능
Meter m = Meter{1.5};
```

`explicit`은 의도치 않은 암시적 변환을 막는 안전 장치다.  
단일 인자 생성자에는 기본적으로 `explicit`을 붙이는 것이 권장된다.

### 변환 연산자

클래스가 다른 타입으로 변환되는 방법을 정의한다.

```cpp
struct Percentage {
    double value;
    explicit operator double() const { return value / 100.0; }
};

Percentage p{75.0};
double d = static_cast<double>(p);  // 0.75
// double d = p;  // explicit이면 에러
```

## C++ 캐스트 4종

### `static_cast`

컴파일 타임에 타입 관계가 명확한 변환. 가장 많이 쓰는 안전한 캐스트.

```cpp
double d = 3.14;
int i = static_cast<int>(d);  // 3

// 상속 관계에서 업캐스트/다운캐스트에도 사용 (다운캐스트는 런타임 검사 없음)
Base* b = new Derived{};
Derived* dp = static_cast<Derived*>(b);
```

### `reinterpret_cast`

비트 수준에서의 재해석. 타입 시스템의 보호를 크게 우회한다.

```cpp
#include <cstdint>

int x = 10;
std::uintptr_t addr = reinterpret_cast<std::uintptr_t>(&x);
```

시스템 코드, FFI, 네트워크 직렬화 등 제한된 영역에서만 사용해야 한다.

### `const_cast`

`const`/`volatile` 속성을 제거하거나 추가한다. const 객체의 const를 제거해 수정하면 UB다.

```cpp
void legacyApi(char* p) {}

void call(const char* str) {
    legacyApi(const_cast<char*>(str));  // C API 호환을 위해 제한적으로 사용
}
```

### `dynamic_cast`

다형 타입에서 **런타임 타입 확인** 후 변환한다. 기반 클래스에 virtual 함수가 있어야 한다.

```cpp
struct Base   { virtual ~Base() = default; };
struct Derived : Base { void hello() { std::cout << "Derived\n"; } };

void call(Base* b) {
    if (auto* d = dynamic_cast<Derived*>(b)) {
        d->hello();  // 변환 성공
    }
    // 변환 실패 시: 포인터 → nullptr 반환, 참조 → std::bad_cast 예외
}
```

## 정적 타입과 동적 타입

#### [코드](./code/정적_동적타입.cpp)

C++은 정적 타입 언어. 변수와 표현식의 타입은 컴파일 타임에 결정된다.

```cpp
Base* p = new Derived{};
// 정적 타입: Base*   (컴파일러가 아는 타입)
// 동적 타입: Derived* (런타임 실제 객체 타입)
```

다형성을 사용하려면 기반 클래스에 `virtual` 함수를 선언해야 한다. `virtual`이 없으면 정적 타입 기준으로 호출된다.

| 캐스트 | 검사 시점 | 실패 시 |
|---|---|---|
| `static_cast` | 컴파일 타임 | 미정의 동작 |
| `dynamic_cast` | 런타임 | `nullptr` 또는 `bad_cast` |
