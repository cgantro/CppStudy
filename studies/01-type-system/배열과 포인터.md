## 배열과 포인터는 다르다

#### [코드](./code/배열과%20포인터.cpp)

```cpp
int arr[3] = {1, 2, 3};
int* p = arr;
```

`arr`의 타입은 `int[3]`이고, `p`의 타입은 `int*`다.

---

## 배열-포인터 변환 (Array Decay)

배열은 대부분의 문맥에서 **첫 번째 원소를 가리키는 포인터로 암시적 변환**된다. 이를 decay라고 한다.

decay가 일어나면 배열의 크기 정보가 소실된다.

```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* p = arr;  // decay: int[5] → int*

std::cout << sizeof(arr) << '\n';  // 20 (5 * 4바이트)
std::cout << sizeof(p) << '\n';    // 8  (포인터 크기, 64비트 기준)
```

함수 인자로 배열을 넘길 때 반드시 decay가 일어난다.

```cpp
// 아래 세 선언은 모두 같다 — 모두 int* 를 받는 함수
void f(int arr[]);
void f(int arr[10]);
void f(int* arr);

// 크기를 같이 넘겨야 안전하게 사용할 수 있다
void f(int* arr, std::size_t n);
```

---

## 포인터 산술

배열 원소 접근은 포인터 산술로 동작한다.

```cpp
int arr[3] = {10, 20, 30};
int* p = arr;

*(p + 0);  // 10
*(p + 1);  // 20
*(p + 2);  // 30
// arr[i]는 *(arr + i)와 동일하다
```

## 더 안전한 대안: `std::array`

C 스타일 배열 대신 `std::array`를 사용하면 크기 정보가 타입에 포함되고 decay가 일어나지 않는다.

```cpp
#include <array>

std::array<int, 3> arr = {1, 2, 3};

std::cout << arr.size() << '\n';  // 3
std::cout << arr.at(1) << '\n';   // 20, 범위 초과 시 예외

// 함수 인자로 넘겨도 크기 정보가 유지됨
void f(const std::array<int, 3>& arr);
```

| | C 스타일 배열 | `std::array` |
|---|---|---|
| 크기 정보 | 함수 전달 시 소실 | 타입에 포함 |
| 범위 검사 | 없음 | `.at()` 사용 시 있음 |
| STL 호환 | 제한적 | 완전 호환 |
