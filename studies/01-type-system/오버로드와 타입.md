# 오버로드

#### [오버로드 코드](./code/overload.cpp)

C++은 함수 이름이 같아도 파라미터 타입이 다르면 다른 함수다.  
호출 시 **overload resolution**이 인자와 후보 함수 집합을 비교해 가장 적합한 함수를 선택한다.

---

## Overload Resolution 단계

후보 함수 중에서 아래 우선순위로 가장 적합한 함수를 고른다. 단계가 낮을수록 변환 비용이 크다.

1. **Exact match** — 타입이 완전히 일치 (cv-qualifier 차이 정도는 허용)
2. **Promotion** — `char` → `int`, `float` → `double` 등 작은 정수/부동소수점 승격
3. **Standard conversion** — `int` → `double`, 파생 → 기반 클래스 포인터 등
4. **User-defined conversion** — 변환 생성자나 변환 연산자 경유
5. **Variadic** — `...` 가변 인자 함수

```cpp
void f(int);
void f(long);

f(10);   // int → exact match: f(int) 호출
f(10L);  // long → exact match: f(long) 호출
```

---

## 모호한 경우 (Ambiguity)

같은 단계에 후보가 둘 이상이면 컴파일 에러가 된다.

```cpp
void f(long);
void f(double);

f(10);  // int → long 변환과 int → double 변환이 동급 → 모호 에러
```

해결 방법: 인자를 명시적으로 캐스팅한다.

```cpp
f(static_cast<long>(10));    // f(long) 호출
f(static_cast<double>(10));  // f(double) 호출
```

## 연산자 오버로드

연산자도 함수처럼 오버로드할 수 있다.

```cpp
struct Vec2 {
    float x, y;

    Vec2 operator+(const Vec2& rhs) const {
        return {x + rhs.x, y + rhs.y};
    }

    bool operator==(const Vec2& rhs) const {
        return x == rhs.x && y == rhs.y;
    }
};

Vec2 a{1, 2}, b{3, 4};
Vec2 c = a + b;  // operator+ 호출
```

스트림 연산자는 멤버 함수로 만들 수 없어 비멤버 함수로 정의한다.

```cpp
std::ostream& operator<<(std::ostream& os, const Vec2& v) {
    return os << '(' << v.x << ", " << v.y << ')';
}

std::cout << a;  // (1, 2)
```

## 템플릿과 오버로드 우선순위

일반 함수와 함수 템플릿이 같은 이름으로 존재하면, **exact match인 일반 함수가 우선**한다.

```cpp
template <typename T>
void f(T) { std::cout << "template\n"; }

void f(int) { std::cout << "non-template\n"; }

f(10);    // non-template (exact match 우선)
f(10.0);  // template (double에 맞는 일반 함수 없음)
```
